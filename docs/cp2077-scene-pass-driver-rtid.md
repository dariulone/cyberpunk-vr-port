Static decode (IDA, raw bytes) of scene-pass driver `sub_1401EC1D0(rcx=passObj, rdx=ctx, r8=arg2)`:

- `[passObj+0x34]` = node count (0 => early-out); `[passObj+0x28]` = node array base, **stride 0x10**.
- **RT-id = byte `ctx+0x38`** — read once at top (`mov r8b,[rdx+0x38]`), copied into a local descriptor (tags `0x09` / dword `0xFD3DDE00`). It is NOT built here; it arrives in the ctx from the caller (the frame-level scene-pass invoker).
- Loop: for each node calls `sub_1401EC404(rcx=node, rdx=ctx, r8=arg2)` — **the same ctx flows into every node executor call**.
- View matrices are NOT in ctx; they are global at `renderer+0x4658` (see [[redengine-render-view-state]] / [[cp2077-node-exec-context-layout]]).

**Why this refines option-C:** the old plan hooked the per-node executor and blind-replayed nodes => WRITE AV ([[cp2077-node-replay-not-viable]]). Cleaner lever: hook the DRIVER `sub_1401EC1D0` (stable address, same render-worker thread as the proven-stable executor hook [[cp2077-framegraph-node-executor-hookable]]). On the hook: (1) call original(passObj, ctx, arg2) = eye-1; (2) swap global view matrices to right-eye; (3) clone ctx with `+0x38` set to a 2nd RT slot; (4) call original again = eye-2; (5) restore. One re-entrant driver call instead of 60 per-node replays.

Both driver `sub_1401EC1D0` and executor `sub_1401EC404` are virtual/registered-render-type dispatched (driver data-refs: `0x142b3a010`/`0x1431fe4d0`/`0x14494980c`); so the frame-level caller can't be reached by a near-call xref — must hook the driver body directly.
